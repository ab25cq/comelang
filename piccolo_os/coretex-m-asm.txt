ARM Cortex-Mのアセンブリ言語について、基本的な内容を簡単に説明します。Cortex-Mは32ビットのRISCプロセッサで、簡潔で効率的な命令セットを持っています。

基本のレジスタ
Cortex-Mには、以下のようなレジスタが用意されています。

汎用レジスタ
R0 ～ R12: 汎用レジスタ
計算やデータの一時保存に使用。
R0～R3は特に関数の引数や戻り値で使用されます。
特殊レジスタ
R13 (SP): スタックポインタ

スタックの現在位置を指します。
Cortex-Mには2つのスタックポインタがあります:
MSP (Main Stack Pointer): リセット後のデフォルト。
PSP (Process Stack Pointer): スレッドモードで使用されます。
R14 (LR): リンクレジスタ

関数から戻るアドレスを保持。
R15 (PC): プログラムカウンタ

次に実行される命令アドレスを指します。
xPSR: プログラムステータスレジスタ

フラグや制御ビットを保持します。
アセンブリ命令の基本
Cortex-Mのアセンブリ命令はシンプルで、以下のような構文を持ちます。

データ操作命令
ロード/ストア

LDR R0, [R1] : R1が指すアドレスから値をロードしてR0に格納。
STR R0, [R1] : R0の値をR1が指すアドレスにストア。
即値のロード

MOV R0, #5 : R0に即値5を格納。
加算・減算

ADD R0, R1, R2 : R1とR2を加算してR0に格納。
SUB R0, R1, #10 : R1から即値10を引いてR0に格納。
論理演算

AND R0, R1, R2 : R1とR2のANDをR0に格納。
ORR R0, R1, R2 : R1とR2のORをR0に格納。
EOR R0, R1, R2 : R1とR2のXORをR0に格納。
制御命令
分岐

B label : 指定したラベルに無条件分岐。
BL label : 指定したラベルに分岐し、戻りアドレスをLRに保存。
条件付き分岐

条件フラグに応じて分岐します。
例:
BNE label : フラグがゼロでない場合に分岐。
BEQ label : フラグがゼロの場合に分岐。
スタック操作
プッシュ/ポップ

PUSH {R0, R1} : R0とR1をスタックに保存。
POP {R0, R1} : スタックから値を取り出してR0とR1に格納。
スタックポインタの変更

SUB SP, SP, #4 : スタックポインタをデクリメント（スタック領域を増加）。
ADD SP, SP, #4 : スタックポインタをインクリメント（スタック領域を減少）。
割り込み/例外
SVC (Supervisor Call)

SVC #0 : ソフトウェア割り込みを呼び出します（システムコールなどに使用）。
割り込み制御

割り込み有効化: CPSIE i
割り込み無効化: CPSID i
簡単な例
以下は、2つの数値を加算して結果をメモリに保存する例です。

asm
コードをコピーする
    LDR R0, =0x20000000   ; メモリアドレスをR0にロード
    MOV R1, #5            ; 即値5をR1にロード
    MOV R2, #10           ; 即値10をR2にロード
    ADD R3, R1, R2        ; R1とR2を加算しR3に格納
    STR R3, [R0]          ; R3の値をR0が指すアドレスにストア
デバッグポイント
Thumbモード

Cortex-Mは常にThumb命令セットを使用。
命令は16ビットまたは32ビット幅。
例外処理

ハードウェアが自動でxPSR、PC、LR、R0-R3を保存。
必要に応じてR4-R11もソフトウェアで保存。
